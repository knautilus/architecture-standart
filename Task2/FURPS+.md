| **Код** | **Требования**                                                                                                               | **Комментарий**               |
|---------|------------------------------------------------------------------------------------------------------------------------------|-------------------------------|
| **F**   | **Функциональные (Functionality)**                                                                                           |                               |
|         | Отображать список доступных депозитов с актуальными ставками на сайте и в интернет-банке                                     |                               |
|         | Шифровать передаваемые с сайта и с интернет-банка данные                                                                     |                               |
|         | Клиент может подать заявку на депозит на сайте, оставив свой номер телефона и Ф.И.О.                                         |                               |
|         | Отображать персонализированные ставки в интернет-банке                                                                       |                               |
|         | Клиент может подать заявку на депозит в интернет-банке, указав счёт и сумму депозита, с подтверждением через СМС-код         |                               |
|         | Менеджеры бэк-офисов депозитов и кредитов должны управлять ставками в АБС                                                    | Перенос данных из Excel в АБС |
|         | Сотружник фронт-офиса должен определять ставку в АБС                                                                         |                               |
|         | Клиент должен получать СМС-уведомления после подтверждения размера ставки и открытия депозита                                |                               |
|         | Клиент может просмотривать список своих депозитов в интернет-банке                                                           | Улучшение UX                  |
| **U**   | **Удобство использования (Usability)**                                                                                       |                               |
|         | Использовать систему дизайна для приложений, которая принята в компании                                                      |                               |
|         | Интерфейс должен быть интуитивно понятным и простым в использовании                                                          |                               |
| **R**   | **Надёжность (Reliability)**                                                                                                 |                               |
|         | Все сервисы должны работать 24/7 и быть доступны в 99,9% случаев                                                             |                               |
|         | В случае сбоя интернет-банк должен переключаться на резервный ЦОД                                                            |                               |
| **P**   | **Производительность (Performance)**                                                                                         |                               |
|         | Отклик по всем операциям должен занимать миллисекунды                                                                        |                               |
|         | Масштабирование: горизонтальное для серверов и ЦОД, вертикальное - для АБС                                                   |                               |
|         | Оптимизировать загрузку справочных данных с помощью кэширования                                                              | Например, с помощью Redis     |
| **S**   | **Поддерживаемость (Supportability)**                                                                                        |                               |
|         | Как можно больше использовать технологии, которые уже есть в банке (например, базы данных MS SQL и Oracle)                   |                               |
|         | Новые технологии должны быть совместимы с существующими платформами разработки, и по ним внутри банка должна быть экспертиза |                               |
|         | Предусмотреть разработку документации для дальнейшего расширения системы                                                     |                               |
| **+R**  | **+ Ограничения (Restrictions)**                                                                                             |                               |
|         | Функционал работы с СМС не должен требовать доработки ядра. Лучше реализовать его силами команды разработки банка            |                               |
|         | АБС может масштабироваться только вертикально из-за своей базы данных                                                        |                               |
|         | Нужно избежать прямой работы интернет-банка с API АБС в новом процессе                                                       |                               |
|         | Использовать Kafka для очередей сообщений в перспективе. Учитывать, что текущая версия интернет-банка несовместима с ней     |                               |
